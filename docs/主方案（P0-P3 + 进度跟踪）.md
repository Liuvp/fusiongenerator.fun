## 二-B、UX 死点击修复（2026-06-08 执行）

> 基于 Clarity 23 天数据分析，修复 3 个死点击高发区域

### ✅ 修复 1：角色槽间 "+" 改为可交互按钮

**问题：** 两个角色槽之间的 "+" 是纯装饰 `<div>`，用户（尤其是移动端）误认为是按钮并多次点击 → 死点击

**修复：** 改为 `<button>`，根据状态提供不同功能：

| 状态 | 行为 | 图标 |
|---|---|---|
| 两个角色都选了 | **交换**角色 1 和角色 2 | ⇄ SWAP |
| 两个角色都没选 | **随机配对** | + RANDOM |
| 只选了一个 | 无操作（button disabled） | + |

**文件：** `components/dragon-ball/fusion-studio.tsx`（CharacterSlot 之间的按钮区域）

### ✅ 修复 2：CharacterSlot 槽位改为可点击

**问题：** 角色槽位的图片区域是 `<div>`，用户点击期望移除角色 → 无反应 → 死点击

**修复：** 槽位图片改为 `<button>`：
- **点击槽位**：仅移除该角色的选择（slot 1 → `setChar1(undefined)`，slot 2 → `setChar2(undefined)`）
- **点击 X 按钮**：清除全部选择（原有行为不变）
- 空槽位：`disabled`，无点击行为
- 添加 `hover:brightness-95 active:scale-95` 视觉反馈

**文件：** `components/dragon-ball/fusion-studio.tsx`（CharacterSlot 组件 + 父组件 `onSlotClick` 传参）

### ✅ 修复 3：AI UploadBox 平板端上传死点击

**问题：** `noClick: true` 禁用原生点击打开文件对话框，`open()` 在部分平板浏览器不可靠 → 死点击

**修复：** 改为动态 `noClick: Boolean(file)`：
- **无文件时**：`noClick: false`，使用原生浏览器点击打开文件对话框（所有设备最可靠）
- **有文件时**：`noClick: true`，由自定义 `handleTap` 接管（显示/隐藏 Replace/Remove 操作面板）

**文件：** `app/ai/components/UploadBox.tsx`

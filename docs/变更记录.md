## 2026-07-16：Auth 优化 + 死点击修复 + FLUX 2 Pro 升级

### 1. Auth 流程优化

**文件：** `app/actions.ts`、`components/dragon-ball/fusion-studio.tsx`

| 改动 | 原来 | 现在 |
|---|---|---|
| 邮箱验证 | 注册后提示"检查邮箱" | 关闭邮箱验证，直接可登录 |
| 注册后行为 | 跳转到登录页手动登录 | 自动登录并跳转到 `/dragon-ball` |
| 成功消息 | "Check your email for a verification link" | "Account created! You can now sign in." |
| Toast 标题 | "Check your email! 📧" | "Account created! 🎉" |
| Google/Email 提示 | "email registration requires verification" | "Both Google and email sign-up are instant." |

**依据：** 用户注册后未收到验证邮件，注册流程中断。关闭邮箱验证 + 自动登录减少摩擦。

### 2. 死点击修复（7 处）

**文件：** `components/dragon-ball/fusion-studio.tsx`、`components/dragon-ball/hero.tsx`

| # | 问题 | 修复 |
|---|---|---|
| 1 | 中间"+"按钮只选1个角色时失效 | 只选1个时点击"+"随机选另一个 |
| 2 | "Start Free Fusion"锚点无反馈 | 改为 `scrollIntoView({ behavior: "smooth" })` |
| 3 | CharacterSlot 嵌套 `<span role="button">` 在 `<button>` 内 | 移到外部，改为独立 `<button>` |
| 4 | "x"清除按钮清空全部角色 | 改为只清除对应槽位 |
| 5 | FUU-SION-HA 按钮 `cursor-help` 看似禁用 | 改为 `cursor-pointer` |
| 6 | 快速点击闭包过期 | 新增 `char1Ref/char2Ref` + sync effect |
| 7 | 移动端嵌套滚动冲突 | `max-h-[320px]` 改为 `md:max-h-[320px]` |

**依据：** Clarity 会话分析发现 3 次以上 dead click，影响用户完成融合。

### 3. FLUX 2 Pro 升级 + 角色参考图

**文件：** `app/api/generate-fusion/route.ts`、`app/api/fusion/generate/route.ts`、`lib/prompt-builder.ts`

| 改动 | 原来 | 现在 |
|---|---|---|
| Dragon Ball 融合 | `fal-ai/flux/dev`（纯文生图） | `fal-ai/flux-2-pro/edit` + 角色参考图 |
| Pokemon/其他 | `fal-ai/flux/dev` | `fal-ai/flux-2-pro`（纯文生图） |
| Negative prompt | 传入 API 参数 | 融入 system prompt（FLUX 2 Pro 不支持） |
| 角色识别度 | 模型靠文字猜 | 模型直接"看到"角色图片 |

**关键代码：**
```ts
// Dragon Ball 融合（有角色图）
result = await fal.run("fal-ai/flux-2-pro/edit", {
    input: {
        prompt: "...融合描述...",
        image_urls: [char1ImageUrl, char2ImageUrl],
        ...
    }
});

// Pokemon/其他（无角色图）
result = await fal.run("fal-ai/flux-2-pro", {
    input: { prompt: fullPrompt, ... }
});
```

**依据：** FLUX 2 Pro（32B 参数 + VLM 编码器）画质提升，角色参考图解决识别度问题。

## 2026-07-01：P0-1 配额墙硬跳转改为内联 Auth Dialog

基于 `clarity-sessions-2026-07-01T06-02-21-816Z.json`（12 条会话）验证。

### P0-1：配额墙 `<Link>` 跳转改为内联 Auth Dialog

**文件：** `components/dragon-ball/fusion-studio.tsx`、`components/pokemon/fusion-studio.tsx`

| 改动 | 原来 | 现在 |
|---|---|---|
| 配额墙按钮行为 | `<Button asChild><Link href="/sign-up?redirect_to=...">` 硬跳转到注册页 | `<Button onClick>` 在当前页弹出 shadcn Dialog |
| 注册/登录切换 | 只有注册页 | Dialog 内支持 Sign Up / Sign In 两种模式切换 |
| Google OAuth | 硬跳转 | Dialog 内嵌 form action，不离开当前页 |
| 邮箱密码注册/登录 | 跳转页面 | Dialog 内嵌表单 + 密码显隐 + 错误内联展示 |
| 退出路径 | 浏览器的返回按钮 | Dialog 右上角 X 关闭，不中断当前流程 |

**依据：** `clarity_5541abbd` 在配额墙点击 "Unlock Unlimited" → 跳转注册页后秒退，说明硬跳转导致用户流失。Inline Dialog 减少上下文切换，提高注册转化率。

- **P0-1 配额墙问题确认：** `clarity_5541abbd`（注册页秒退）证实硬跳转问题仍在
